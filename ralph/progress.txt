## Codebase Patterns
- All UI is built programmatically in `_build_ui()` — no .tscn files
- Test runners extend `SceneTree`, use `_initialize()` entry point, `_failures` array, `_assert()` helper, and `_finish()` that prints OK token + `quit(0)` or errors + `quit(1)`
- Static utility classes extend `RefCounted` and use `class_name` for global access
- Test runners must `preload()` scripts and call static methods on the preloaded const — `class_name` globals aren't available in headless `--script` mode
- Explicitly type variables receiving static method returns to avoid GDScript type inference errors
- Expose computation logic as `static func` for testability
- Modal dialogs: ColorRect backdrop + CenterContainer + PanelContainer, emit signals for confirm/cancel
- Multi-select chip filters: Button with `toggle_mode = true`, `toggled` signal with `.bind(id)`, OR logic within chip groups
- CardDisplayComponent: set `custom_minimum_size` and `size` AFTER `apply_card()` (which overrides them); use `set_anchors_and_offsets_preset(PRESET_FULL_RECT)` after `add_child` for responsive sizing
- Card grid uses wrapper Control with MOUSE_FILTER_STOP + gui_input for click handling (CardDisplayComponent has MOUSE_FILTER_IGNORE on all children)
- Navigation controller pattern: parent swaps child screens via signals; keep list screen alive (hidden), recreate editor each time
- All scripts that use class_name globals must add preload() constants for headless --script mode compatibility
- DeckPersistence stores decks as JSON at `user://decks/<sanitized_name>.json`
- Compact deck list pattern: scrollable VBoxContainer with rows sorted by cost ascending, cost badge + card name + quantity
- Godot binary is at `/Applications/Godot.app/Contents/MacOS/Godot`
- ArenaDraftEngine: static utility class at `src/arena/arena_draft_engine.gd` — rarity rolls, card pool filtering, pick options with fallback
- CardCatalog.load_default() returns `{cards, card_by_id, error}` — use `card_by_id` as the card database dictionary for ArenaDraftEngine
- ArenaRunManager persistence: save_run() after state changes, clear_run() on completion/abandon; JSON at `user://arena/run.json`
- ArenaRunManager: non-static class holding run state; State enum tracks lifecycle; boss is match 9; deck grows via post-win picks
- Self-referencing class_name in static methods (e.g. `ArenaRunManager.new()` inside ArenaRunManager) breaks headless mode; use `load("res://path/to/script.gd").new()` instead
